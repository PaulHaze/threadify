This is the very first draft of a prd for Threadify. Threadify is going to be a simple CLI/terminal app at first and then a mac based GUI app (with the potential to expand it further). 

THE INSPIRATION:
I often come across reddit threads with a lot of music recommendations in various styles or genres (house, ambient, jazz etc). Oftentimes I will go through the thread and manually pull out the songs or albums mentioned into a custom playlist. I'd like to automate this using AI.

THE GOAL:
The final version of the APP will be GUI on the mac (as thats what I have, linux and PC versions possibly to come, maybe as a simple web wrapper) where you paste in the URL of the reddit thread (or any HTML page with a list of music recommendations). The app will extract what it thinks is the relevant albums and songs, will output a split screen list/columnof what it found from the text on the right, with the original highlighted text on the left (so the user can double check and see if it got it right). This information can be edited in the right window to correct any AI errors (like incorrect artist/song/album fields etc). The app will then create a spotify playlist in the users account with a name and description of the users choosing and find and add all the songs from the reddit thread/html page.

An early version will be just triggered from the command line, maybe in a 2 stage process where a command like 'read_thread {url}' which will parse the thread into a JSON file and then 'create_playlist {parsed.JSON} {playlist name}' will go ahead make the playlist. 

CONSIDERATIONS:
- This is a brand new project with no real constraints on what code/language/framework can be used. It will be being built on a mac so mac friendly code is the main consideration. Cross platform is the desired end goal.
- Hallucinations/misreads will be an issue so some sort of user friendly way of letting them check the output before it gets sent to spotify, and again with the spotify result before creating the playlist would be good. 
- This will be a self-contained and open source project. The user will need to provide their own spotify access authorization and LLM key for parsing the JSON. I will walk through how they can do that in the read me.
